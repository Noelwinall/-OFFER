# HK Edu App - 產品設計文件

## 設計原則

**目標用戶**: 香港家長（零基礎選校）  
**核心價值**: 把混亂的想法 → 結構化篩選條件 → 清晰的學校推薦  
**設計風格**: 克制、高級、工具感（不是政府網站、不是 Excel 列表）  
**平台**: iOS 原生體驗（遵循 Apple HIG）  
**方向**: 直向 9:16，單手操作優先

---

## 畫面結構

### 1. 首頁（問答引導模式入口）
**主要內容**:
- 頂部標題：「找到最適合孩子的學校」
- 副標題：「5 個問題，精準推薦」
- 主按鈕：「開始選校」（CTA）
- 次要入口：「瀏覽所有學校」（傳統檢索）、「探索學校」（滑卡模式）

**功能**:
- 點擊「開始選校」→ 進入問答引導流程
- 點擊「瀏覽所有學校」→ 進入學校列表頁
- 點擊「探索學校」→ 進入滑卡模式

---

### 2. 問答引導流程（5-6 個問題）
**畫面佈局**:
- 頂部進度條（顯示當前問題進度）
- 中央問題卡片（大字體問題）
- 選項列表（單選/多選按鈕）
- 底部：「不確定」按鈕 + 「下一步」按鈕

**問題清單**:
1. **學段**: 幼稚園 / 小學 / 中學
2. **地區**: 港島 / 九龍 / 新界 / 不限
3. **學費預算**: 免費 / $20,000 以下 / $20,000-$50,000 / $50,000-$100,000 / $100,000 以上 / 不限
4. **學校類型**: 國際 / 資助 / 直資 / 私立 / 公立 / 不限
5. **課程取向**: IB / DSE / IGCSE / A-Level / 其他 / 不限
6. **英文環境**: 全英文 / 中英雙語 / 以中文為主 / 不限

**互動邏輯**:
- 每題允許「不確定」（跳過該篩選條件）
- 最後一題完成後 → 顯示「查看推薦」按鈕
- 點擊「查看推薦」→ 進入推薦結果頁

---

### 3. 推薦結果頁
**主要內容**:
- 頂部：「為您推薦 X 所學校」
- 學校卡片列表（使用統一卡片組件）
- 底部：「重新篩選」按鈕

**學校卡片內容**:
- 學校名稱
- 學校類型標籤（Tag）
- 地區
- 學費區間
- 2-3 條亮點 snapshot（短句）
  - 示例：「提供 IB 課程體系」、「英文教學環境為主」、「面試偏重表達能力」
- 右上角：收藏按鈕（愛心圖標）

**互動邏輯**:
- 點擊卡片 → 進入學校詳情頁
- 點擊收藏按鈕 → 加入收藏列表（LocalStorage）
- 點擊「重新篩選」→ 返回問答引導流程

---

### 4. 傳統檢索模式（學校列表頁）
**主要內容**:
- 頂部搜尋框（支援學校名稱搜尋）
- 篩選按鈕（學校類型、地區、課程、學費）
- 學校卡片列表（使用統一卡片組件）

**篩選邏輯**:
- 點擊篩選按鈕 → 彈出篩選面板（Bottom Sheet）
- 支援多條件組合篩選
- 即時更新列表

---

### 5. 學校詳情頁
**主要內容**:
- 頂部：學校名稱 + 收藏按鈕
- 基本資訊區塊：
  - 學校類型、地區、學費
  - 課程體系
  - 聯絡方式（電話、地址）
- 申請資訊區塊：
  - 申請材料清單
  - 申請連結（可複製）
- 亮點描述（3-5 條）
- 底部：「加入對比」按鈕

**互動邏輯**:
- 點擊收藏按鈕 → 加入/移除收藏
- 點擊「加入對比」→ 加入對比列表（最多 3 所）
- 點擊申請連結 → 複製到剪貼簿

---

### 6. 收藏頁
**主要內容**:
- 頂部：「我的收藏」
- 學校卡片列表（使用統一卡片組件）
- 底部：「對比學校」按鈕（顯示已選數量）

**互動邏輯**:
- 點擊卡片 → 進入學校詳情頁
- 點擊「對比學校」→ 進入對比頁

---

### 7. 對比頁
**主要內容**:
- 頂部：「學校對比（X/3）」
- 橫向滾動卡片（最多 3 所）
- 對比表格（學校類型、地區、學費、課程、亮點）
- 底部：「列印對比」按鈕

**互動邏輯**:
- 點擊「列印對比」→ 生成 PDF 或分享

---

### 8. 滑卡探索模式
**主要內容**:
- 中央大卡片（學校資訊）
- 左滑跳過 / 右滑收藏
- 頂部：剩餘滑動次數（如「今日剩餘 25 次」）
- 底部：「返回首頁」按鈕

**互動邏輯**:
- 左滑 → 跳過該學校
- 右滑 → 加入收藏
- 達到每日限制 → 顯示「今日已達上限」提示

---

## 統一學校卡片組件規範

**卡片尺寸**: 寬度 100%（左右留 16px 邊距），高度自適應  
**圓角**: 12px  
**陰影**: 淺陰影（elevation 2）  
**內邊距**: 16px

**內容結構**:
1. 學校名稱（18px，粗體）
2. 學校類型標籤（Tag，12px，圓角 4px）
3. 地區 + 學費區間（14px，灰色）
4. 亮點 snapshot（14px，行高 1.5）
   - 使用圓點符號（•）分隔
   - 最多顯示 3 條
5. 右上角收藏按鈕（愛心圖標）

**文字顏色**:
- Foreground: `#11181C`（淺色模式）/ `#ECEDEE`（深色模式）

- Success: `#22C55E`（綠色）
- Warning: `#F59E0B`（橙色）

## 核心用戶流程

### 流程 1: 問答引導 → 推薦結果 → 學校詳情
1. 用戶點擊首頁「開始選校」
2. 依序回答 5-6 個問題
3. 查看推薦結果（學校卡片列表）
4. 點擊卡片查看學校詳情
5. 點擊「加入收藏」或「加入對比」

### 流程 2: 傳統檢索 → 學校詳情
1. 用戶點擊首頁「瀏覽所有學校」
2. 使用搜尋框或篩選條件
3. 點擊卡片查看學校詳情

### 流程 3: 滑卡探索 → 收藏
1. 用戶點擊首頁「探索學校」
2. 左滑跳過 / 右滑收藏
3. 達到每日限制後返回首頁

### 流程 4: 收藏 → 對比 → 列印
1. 用戶進入「收藏頁」
2. 選擇最多 3 所學校
3. 點擊「對比學校」
4. 查看對比表格
5. 點擊「列印對比」生成 PDF

---

## 技術實現要點

### 資料儲存
- **LocalStorage**: 收藏列表、對比列表、滑卡進度
- **不使用後端**: 當前階段純前端實現

### 資料來源
- CSV 匯入學校資料
- 核心欄位：學校名稱、類型、地區、學費、課程、亮點描述
- 軟性描述通過規則/模板半自動生成

### 免責聲明
- 每個頁面底部顯示：「資訊基於公開資料整理，僅供參考，以學校官方為準」

## 風險與取捨

### 風險
- **資料準確性**: 依賴 CSV 匯入，需確保核心欄位準確
- **軟性描述生成**: 需設計合理的規則/模板，避免過度吹噓

### 取捨
- **不做複雜推薦演算法**: 當前階段基於規則篩選，不做機器學習
- **不做用戶帳號系統**: 使用 LocalStorage，不做雲端同步
- **滑卡模式簡化**: 不做個性化推薦，隨機展示學校

---

## 產品判斷

1. **問答引導是核心**: 這是產品的差異化優勢，必須做到流暢、直覺
2. **學校卡片是基礎**: 統一的卡片體系確保一致性，提升專業感
3. **資料真實性優先**: 不造假、不吹噓，明確標註資料來源
4. **滑卡模式是加分項**: 增強探索感，但不是核心功能，可最小實現

---

## 總結

HK Edu App 的核心是**問答引導模式**，通過結構化問題幫助零基礎家長快速找到合適學校。產品設計遵循 iOS 原生體驗，追求克制、高級、工具感。統一的學校卡片體系確保一致性，收藏與對比功能提升實用性。當前階段優先完成 MVP 核心功能，確保產品可立即上線。
